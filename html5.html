Introduction to HTML

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It defines the structure and content of a webpage.

What is HTML?

HTML stands for HyperText Markup Language. It is the standard markup language used to create web pages. HTML provides the structure of a webpage, which is then enhanced and modified by other technologies like CSS and JavaScript.

Key points about HTML:

  • HTML is not a programming language; it is a markup language
  • HTML consists of a series of elements/tags
  • HTML elements tell the browser how to display content
  • HTML documents are also called web pages

HTML vs HTML5

HTML5 is the latest version of HTML, which introduces many new features and improvements over HTML4.

Key Differences

Feature HTML4 HTML5
Doctype Declaration Long and complex Simple: <!DOCTYPE html>
Character Encoding Complex declaration Simple: <meta charset="UTF-8">
Semantic Elements Limited (<div> heavily used) Many new tags: <header>, <nav>, <section>, etc.
Form Controls Basic Enhanced with new input types: email, date, time, color, etc.
Multimedia Required plugins Native support with <audio> and <video> tags
Graphics Limited to images Added <canvas> and <svg> for advanced graphics
Storage Cookies only Added localStorage, sessionStorage, and IndexedDB

Setting Up

To get started with HTML, you need very little:

  • A text editor (e.g., Visual Studio Code, Sublime Text, Notepad++)
  • A web browser (e.g., Chrome, Firefox, Safari)

Creating Your First HTML Page

  1. Open your text editor
  2. Create a new file and save it with a .html extension (e.g., index.html)
  3. Write your HTML code
  4. Save the file
  5. Open the file in a web browser

Basic Template

Here's a basic HTML template to get you started:

                
<!DOCTYPE html>
<html lang="en">
    <head>
        <meta charset="UTF-8">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>My First HTML Page</title>
    </head>
    <body>
        <h1>Hello, World!</h1>
        <p>This is my first HTML page.</p>
    </body>
</html>

HTML Basics

Understanding the fundamental building blocks of HTML documents.

Document Structure

Every HTML document has a required structure that includes the following declarations and elements:

DOCTYPE Declaration

The DOCTYPE declaration is not an HTML tag; it is an instruction to the web browser about what version of HTML the page is written in. 

<!DOCTYPE html>

HTML Element

The <html> element is the root element of an HTML page. All other elements are contained within it. 

<html lang="en">
    <!-- All your HTML content goes here -->
</html>

HEAD Element

The <head> element contains meta-information about the document, such as its title, scripts, and links to stylesheets. 

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>

TITLE Element

The <title> element defines the title of the document, which appears in the browser's title bar or tab. 

<title>My Web Page</title>

BODY Element

The <body> element contains all the content that is displayed on a web page, including text, images, links, and more. 

<body>
    <h1>This is a heading</h1>
    <p>This is a paragraph.</p>
</body>

Note

The structure of an HTML document is hierarchical. The <html> element contains the <head> and <body> elements. The <head> contains document metadata, while the <body> contains the visible content.

Elements

HTML elements are the building blocks of HTML pages. Elements are represented by tags, and they define various content types and their structure.

Anatomy of an HTML Element

A typical HTML element consists of:

  • Opening tag: <tagname>
  • Content: The information between the opening and closing tags
  • Closing tag: </tagname>

Nesting HTML Elements

HTML elements can be nested inside other elements to create a hierarchical structure. 

<div>
    <h1>Main Heading</h1>
    <p>This paragraph is inside a div element.</p>
</div>

Empty Elements

Some HTML elements have no content and therefore no closing tag. These are called empty elements or self-closing tags.

Examples:

<img src="image.jpg" alt="An image">
<br>
<hr>
<input type="text">
<meta charset="UTF-8">

In HTML5, the trailing forward slash in self-closing tags is optional: 

<!-- These are equivalent in HTML5 -->
<img src="image.jpg" alt="An image">
<img src="image.jpg" alt="An image" />

Attributes

HTML attributes provide additional information about an element. They are always specified in the opening tag and usually come in name/value pairs.

Attribute Syntax

<element attribute="value">Content</element>

Common Attributes

Attribute Purpose Example
class Specifies one or more class names for an element <div class="container main">
id Specifies a unique id for an element <div id="header">
src Specifies the URL of an external resource <img src="image.jpg">
href Specifies the URL of a linked resource <a href="https://example.com">
alt Provides alternative text for image elements <img src="logo.png" alt="Company Logo">
style Specifies inline CSS styles for an element <p style="color: blue;">
title Provides additional information (tooltip text) <abbr title="HyperText Markup Language">HTML</abbr>

Boolean Attributes

Some attributes are boolean, meaning they can be present or absent, but don't have a value assigned to them. 

<!-- These are equivalent -->
<input type="checkbox" checked>
<input type="checkbox" checked="checked">

Important

In HTML, attribute names are not case-sensitive, but it's a best practice to use lowercase. Values should be enclosed in double quotes, though single quotes can also be used.

Inline Styles

The style attribute allows you to add CSS styles directly to an HTML element. While separating content (HTML) from presentation (CSS) is generally recommended, inline styles can be useful for quick styling or testing.

Basic Syntax

<element style="property1: value1; property2: value2;">Content</element>

Examples

<!-- Text color and font size -->
<p style="color: red; font-size: 20px;">This is a red paragraph with larger text.</p>

<!-- Background color and text alignment -->
<div style="background-color: #f0f0f0; text-align: center; padding: 10px;">
    Centered text in a gray box
</div>

<!-- Multiple styling properties -->
<h2 style="color: #003366; font-family: Arial, sans-serif; text-decoration: underline; margin-bottom: 15px;">
    Styled Heading
</h2>

Note

While inline styles work, they're generally not recommended for production websites because:

  • They mix content with presentation
  • They make maintenance harder
  • They can't be easily reused across multiple elements
  • They override external stylesheets due to their high specificity

For larger projects, use external CSS files instead.

Text Content

HTML provides various elements to structure and format text content on web pages.

Headings

HTML offers six levels of headings, from <h1> (most important) to <h6> (least important). Headings are block-level elements and automatically add space before and after the content.

Heading Elements

<h1>This is Heading 1</h1>
<h2>This is Heading 2</h2>
<h3>This is Heading 3</h3>
<h4>This is Heading 4</h4>
<h5>This is Heading 5</h5>
<h6>This is Heading 6</h6>

This is Heading 1

This is Heading 2

This is Heading 3

This is Heading 4

This is Heading 5
This is Heading 6

Best Practices for Headings

  • Use <h1> for the main heading of your page (typically once per page)
  • Structure headings hierarchically (h1, then h2, then h3, etc.)
  • Don't skip heading levels for proper document outline
  • Headings are important for accessibility and SEO

Paragraphs

The <p> element defines a paragraph in HTML. Browsers automatically add some margin before and after each paragraph.

Basic Usage

<p>This is a paragraph of text. Paragraphs are block-level elements in HTML.</p>
<p>This is another paragraph. Notice how browsers add space between paragraphs.</p>

Formatting Within Paragraphs

You can use various inline elements within paragraphs to format text: 

<p>
  This text is normal, <b>this text is bold</b>, and <i>this text is italic</i>.
  You can also use <em>emphasis</em> or <strong>strong importance</strong>.
  This text contains a <br> line break.
</p>

Lists

HTML provides three types of lists to organize content: unordered lists, ordered lists, and definition lists.

Unordered Lists

Unordered lists (<ul>) display items with bullet points. Each list item is defined with the <li> tag. 

<ul>
  <li>Coffee</li>
  <li>Tea</li>
  <li>Milk</li>
</ul>
  • Coffee
  • Tea
  • Milk

Ordered Lists

Ordered lists (<ol>) display items with numbers. Each list item is defined with the <li> tag. 

<ol>
  <li>First step</li>
  <li>Second step</li>
  <li>Third step</li>
</ol>
  1. First step
  2. Second step
  3. Third step

Definition Lists

Definition lists (<dl>) display items as term-definition pairs, useful for glossaries or metadata. 

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>
HTML
HyperText Markup Language
CSS
Cascading Style Sheets

Media & Hyperlinks

Learn how to incorporate links, images, and tables into your HTML documents.

Links

Links, or hyperlinks, allow users to navigate between web pages. In HTML, links are created using the <a> (anchor) element with the href attribute.

Basic Link Syntax

<a href="url">Link text</a>

Types of Links

Type Example Description
Absolute URL <a href="https://www.example.com">Example Website</a> Links to an external resource with full URL
Relative URL <a href="about.html">About Us</a> Links to a page within the same website
Email Link <a href="mailto:info@example.com">Email Us</a> Opens the user's email client
Phone Link <a href="tel:+1234567890">Call Us</a> Initiates a phone call on mobile devices
ID Link (Bookmark) <a href="#section1">Go to Section 1</a> Links to an element with a specific ID on the same page

Link Attributes

Attribute Value Description
target _blank Opens the link in a new tab/window
title Text Displays tooltip text on hover
download Filename or empty Suggests downloading the link instead of navigating
rel nofollow, noopener, etc. Specifies relationship between current and linked document

Examples

<!-- External link that opens in a new tab -->
<a href="https://www.example.com" target="_blank" rel="noopener">Visit Example.com</a>

<!-- Link to an email address with subject line -->
<a href="mailto:contact@example.com?subject=Website%20Inquiry">Email Us</a>

<!-- Link to a section on the same page -->
<a href="#contact">Contact Section</a>
...
<section id="contact">
<h2>Contact Us</h2>
...
</section>

Images

The <img> element is used to embed images in an HTML document. It's an empty element (has no closing tag) and requires the src attribute to specify the image URL.

Basic Image Syntax

<img src="image_url" alt="alternative text">

Important Attributes

Attribute Description Example
src Required. Specifies the path to the image src="images/logo.png"
alt Required. Provides alternative text for screen readers and if the image cannot load alt="Company Logo"
width Specifies the width of the image in pixels width="300"
height Specifies the height of the image in pixels height="200"
loading Specifies how the browser should load the image loading="lazy"

Examples

<!-- Basic image -->
<img src="cat.jpg" alt="A cat sitting on a windowsill">

<!-- Image with specified dimensions -->
<img src="logo.png" alt="Company Logo" width="200" height="100">

<!-- Responsive image that loads lazily -->
<img src="banner.jpg" alt="Website banner" width="100%" loading="lazy">

<!-- Image with a link -->
<a href="product.html">
<img src="product-thumbnail.jpg" alt="Product thumbnail">
</a>

Best Practices for Images

  • Always include the alt attribute for accessibility
  • Specify image dimensions to help the browser allocate space properly
  • Use appropriate image formats (JPG for photos, PNG for graphics with transparency, SVG for icons and logos)
  • Optimize images for web to reduce file size and improve loading speed
  • Consider using the loading="lazy" attribute for images that are not in the initial viewport

Tables

HTML tables are used to display data in rows and columns. While they should not be used for layout purposes (use CSS for that), tables are excellent for presenting tabular data.

Basic Table Structure

<table>

Table Elements

<tr> Defines a row in a table

Document Organization

Learn how to organize and structure your HTML documents with divs, classes, IDs, and semantic elements.

Divs

The <div> element is a generic container for grouping content. It has no semantic meaning but is useful for styling purposes with CSS or for organizing content into logical blocks.

Basic Usage

<div>
    <h2>Section Title</h2>
    <p>This is a paragraph inside a div.</p>
    <p>This is another paragraph inside the same div.</p>
</div>

Nested Divs

Divs can be nested to create more complex layouts:

<div class="container">
    <div class="header">
        <h1>Website Title</h1>
    </div>

    <div class="content">
        <div class="sidebar">
            <ul>
                <li>Menu Item 1</li>
                <li>Menu Item 2</li>
            </ul>
        </div>
        <div class="main">
            <h2>Article Title</h2>
            <p>Article content goes here...</p>
        </div>
        <div class="footer">
          <p>Copyright 2025</p>
        </div>
    </div>
</div>

Important Note

While <div> elements are very useful, they should not be overused. In HTML5, it's better to use semantic elements (like <header>, <nav>, <section>, etc.) whenever possible as they provide meaning to both browsers and developers.

Classes

The class attribute is used to specify one or more class names for an HTML element. Classes allow you to define styles for elements with the specified class name through CSS.

Basic Syntax

<element class="classname">Content</element>

Multiple Classes

An element can have multiple classes, separated by spaces:

<div class="container centered highlight">
  This div has three classes: container, centered, and highlight.
</div>

Examples

<!-- Elements with the same class -->
<p class="important">This is an important paragraph.</p>
<p>This is a regular paragraph.</p>
<p class="important">This is another important paragraph.</p>

<!-- Different elements with the same class -->
<h2 class="highlight">Highlighted Heading</h2>
<p class="highlight">Highlighted paragraph.</p>

<!-- Elements with multiple classes -->
<div class="card shadow rounded">
  <h3>Card Title</h3>
  <p>Card content...</p>
</div>

CSS and Classes

Classes are particularly useful when styling with CSS. You can target elements with a specific class using the dot selector: 

.important {
  color: red;
  font-weight: bold;
}

.highlight {
  background-color: yellow;
}

.card {
  border: 1px solid #ccc;
  padding: 10px;
}

.shadow {
  box-shadow: 0 4px 8px rgba(0,0,0,0.1);
}

.rounded {
  border-radius: 8px;
}

IDs

The id attribute is used to specify a unique identifier for an HTML element. Unlike classes, IDs must be unique within a document.

Basic Syntax

<element id="uniqueid">Content</element>

Primary Uses for IDs

  1. Linking to a specific section of a page 

    <!-- Link to the section -->
<a href="#contact">Go to Contact Section</a>

<!-- The section with the ID -->
<section id="contact">
  <h2>Contact Us</h2>
  <p>Contact information and form...</p>
</section>

  2. JavaScript manipulation

    <button id="submitBtn">Submit</button>

<script>
  document.getElementById('submitBtn').addEventListener('click', function() {
    alert('Button clicked!');
  });
</script>

  3. Styling specific elements with CSS

    <div id="main-header">Website Header</div>

<style>
  #main-header {
    background-color: #333;
    color: white;
    padding: 10px;
  }
</style>

ID vs Class

Feature ID Class
Uniqueness Must be unique in a document Can be used on multiple elements
Usage per element An element can have only one ID An element can have multiple classes
CSS selector #id_name .class_name
JavaScript access getElementById() getElementsByClassName()
CSS specificity Higher (more specific) Lower (less specific)
URL targeting Can be targeted with #id_name in URLs Cannot be targeted in URLs

Best Practices

  • Use IDs sparingly and only when necessary
  • Prefer classes for styling multiple elements
  • Ensure IDs are unique within a document
  • Use meaningful, descriptive names for IDs
  • ID names should not start with numbers and should not contain spaces

Semantic HTML

Semantic HTML uses HTML elements that clearly describe their meaning to both the browser and the developer. HTML5 introduced many semantic elements to better structure web documents.

Benefits of Semantic HTML

  • Improved accessibility for screen readers and assistive technologies
  • Better SEO (Search Engine Optimization)
  • Easier code maintenance and readability
  • Clearer document structure

Common Semantic Elements

Element Description
<header> Represents introductory content, typically a group of navigational aids or introductory information
<nav> Defines a section of navigation links
<main> Specifies the main content of a document (should be unique to the document)
<article> Represents a self-contained composition in a document, which could be independently distributable or reusable
<section> Defines a thematic grouping of content, typically with a heading
<aside> Represents content that is tangentially related to the content around it (like a sidebar)
<footer> Defines a footer for a document or section, typically containing authorship, copyright, or contact information
<figure> Represents self-contained content, like illustrations, diagrams, photos, code listings, etc.
<figcaption> Defines a caption or legend for a figure
<time> Represents a specific period in time or a date
<mark> Represents text that is highlighted or marked for reference

Example of Semantic HTML Structure

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Semantic HTML Example</title>
</head>
<body>
  <header>
    <h1>My Website</h1>
    <nav>
      <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
        <li><a href="/contact">Contact</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <section>
      <h2>Featured Articles</h2>
      <article>
        <header>
          <h3>Article Title</h3>
          <p>Published on <time datetime="2025-02-14">February 14, 2025</time></p>
        </header>
        <p>Article content goes here...</p>
        <figure>
          <img src="image.jpg" alt="Descriptive image text">
          <figcaption>Caption for the image</figcaption>
        </figure>
      </article>
    </section>

    <aside>
      <h3>Related Links</h3>
      <ul>
        <li><a href="#">Related Article 1</a></li>
        <li><a href="#">Related Article 2</a></li>
      </ul>
    </aside>
  </main>

  <footer>
    <p>© 2025 My Website. All rights reserved.</p>
  </footer>
</body>
</html>

  </article>

Forms

HTML forms are used to collect user input. They contain various input elements like text fields, checkboxes, radio buttons, and more.

Form Basics

HTML forms are created using the <form> element, which contains various input elements and buttons for user interaction.

Basic Form Structure

<form action="/submit-form" method="post">
  <!-- Form elements go here -->
  <input type="text" name="username" placeholder="Username">
  <input type="password" name="password" placeholder="Password">
  <input type="submit" value="Login">
</form>

Form Process Flow

  1. The user fills out the form with data
  2. The user submits the form by clicking a submit button or pressing Enter
  3. The browser sends the form data to the server specified in the action attribute
  4. The server processes the data using the method specified in the method attribute
  5. The server sends a response back to the browser, typically a new page or a redirect

Form Attributes

The <form> element can have several attributes that control how the form behaves.

Attribute Description Example
action Specifies where to send the form data when the form is submitted action="/submit-form"
method Specifies the HTTP method to use when sending form data (get or post) method="post"
enctype Specifies how form data should be encoded when submitted (used with method="post") enctype="multipart/form-data"
autocomplete Specifies whether the browser should autocomplete form fields autocomplete="off"
novalidate Specifies that the form should not be validated when submitted novalidate
target Specifies where to display the response after submitting the form target="_blank"
name Specifies a name for the form (for DOM access) name="loginForm"

Examples

<!-- Basic form with post method -->
<form action="/submit" method="post">
  <!-- Form elements -->
</form>

<!-- Form with file uploads -->
<form action="/upload" method="post" enctype="multipart/form-data">
  <input type="file" name="document">
  <input type="submit" value="Upload">
</form>

<!-- Form with autocomplete disabled -->
<form action="/login" method="post" autocomplete="off">
  <input type="text" name="username">
  <input type="password" name="password">
  <input type="submit" value="Login">
</form>

<!-- Form that opens response in a new tab -->
<form action="/search" method="get" target="_blank">
  <input type="text" name="query">
  <input type="submit" value="Search">
</form>

GET vs. POST

Feature GET POST
Visibility Form data is visible in the URL Form data is not visible in the URL
Bookmarking Can be bookmarked Cannot be bookmarked
Data size Limited (about 2048 characters) Unlimited
Security Less secure (data in URL) More secure (data in request body)
Cache Can be cached Not cached
Best for Retrieving data (searches, filters) Submitting data (logins, uploads, forms)

Form Elements

HTML provides various form elements to capture different types of user input.

Common Form Elements

Element Description Example
<input> Creates various input controls based on its type attribute <input type="text" name="username">
<label> Defines a label for a form element, improving accessibility <label for="username">Username:</label>
<select> Creates a dropdown list <select name="country"><option>USA</option></select>
<option> Defines an option in a select list <option value="us">United States</option>
<textarea> Creates a multi-line text input <textarea name="message" rows="4"></textarea>
<button> Creates a clickable button <button type="submit">Send</button>
<fieldset> Groups related form elements <fieldset><legend>Personal Info</legend>...</fieldset>
<legend> Defines a caption for a fieldset <legend>Contact Information</legend>
<datalist> Provides a list of predefined options for an input <datalist id="browsers"><option value="Chrome"></datalist>
<output> Represents the result of a calculation <output name="result">0</output>

Examples

<!-- Label and input field -->
<label for="fullname">Full Name:</label>
<input type="text" id="fullname" name="fullname">

<!-- Textarea for comments -->

Browser Compatibility

While all modern browsers support HTML5 input types, older browsers may not. When an HTML5 input type is not supported, the browser will treat it as a regular text input.

This degradation is actually beneficial as it ensures forms remain usable across all browsers, even if the enhanced functionality isn't available.

Best Practices

Following these HTML best practices will help you create robust, accessible, and maintainable web pages.

Document Structure

  • Always declare the document type with <!DOCTYPE html>
  • Specify the language attribute in the html tag: <html lang="en">
  • Use UTF-8 character encoding: <meta charset="UTF-8">
  • Include the viewport meta tag for responsive design: <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Provide a descriptive page title: <title>Page Title | Site Name</title>

Semantic HTML

  • Use HTML5 semantic elements (<header>, <nav>, <main>, etc.) instead of generic <div> elements whenever appropriate
  • Use heading elements (<h1> to <h6>) in a hierarchical order
  • Use <button> for clickable buttons instead of styled <div> or <a> elements
  • Use <a> elements for navigation, not for buttons that perform actions
  • Use <ul> or <ol> for lists of items
  • Use <table> only for tabular data, not for layout

Accessibility

  • Use appropriate alt attributes for images
  • Associate <label> elements with form inputs
  • Use ARIA (Accessible Rich Internet Applications) attributes when necessary
  • Ensure keyboard navigation works for all interactive elements
  • Maintain sufficient color contrast for text
  • Use appropriate heading hierarchy
  • Provide captions or transcripts for multimedia content

Performance

  • Minimize HTTP requests by combining CSS and JavaScript files
  • Optimize image sizes and use appropriate formats
  • Use lazy loading for images: <img loading="lazy" src="image.jpg" alt="Description">
  • Place CSS in the head and JavaScript before the closing body tag
  • Specify image dimensions to help the browser allocate space before the image loads

Coding Style

  • Use lowercase for HTML element names, attributes, and values
  • Use double quotes for attribute values
  • Indent nested elements for better readability
  • Close all elements properly, even self-closing elements in XHTML or XML style if preferred: <img src="image.jpg" alt="Description" />
  • Use meaningful names for IDs and classes
  • Keep markup clean and minimal

Validation

  • Validate your HTML using the W3C Validator (https://validator.w3.org/)
  • Ensure your HTML is well-formed and follows standards
  • Fix validation errors and warnings

SEO

  • Use descriptive, keyword-rich page titles
  • Include meta description tags
  • Use heading elements appropriately
  • Provide descriptive text for links instead of generic phrases like "click here"
  • Use semantic markup that helps search engines understand your content
  • Include structured data when appropriate (using schema.org vocabulary)

HTML5 vs. HTML4: Key Differences

HTML5 introduced many improvements over HTML4:

  1. Simplified DOCTYPE declaration: HTML5 uses <!DOCTYPE html> instead of the lengthy DOCTYPE in HTML4.
  2. New semantic elements: HTML5 added elements like <header>, <nav>, <section>, etc., for better document structure.
  3. Multimedia support: Native support for audio and video with <audio> and <video> elements.
  4. Graphics capabilities: Added <canvas> and <svg> for drawing and vector graphics.
  5. Form enhancements: New input types, form validation, and elements like <datalist>.
  6. New APIs: Added APIs for geolocation, drag-and-drop, web storage, etc.
  7. Mobile-friendly: Better support for responsive design and mobile devices.
  8. Improved storage options: Added localStorage, sessionStorage, and IndexedDB.
  9. Better error handling: More forgiving parsing algorithm.
  10. Removed deprecated elements: Eliminated presentational elements like <font> and <center>.
