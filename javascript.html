Introduction to JavaScript

JavaScript is a lightweight, interpreted, or just-in-time compiled programming language with first-class functions. While it is most well-known as the scripting language for Web pages, it's used in many non-browser environments as well. JavaScript is a prototype-based, multi-paradigm, single-threaded, dynamic language, supporting object-oriented, imperative, and declarative styles.

What is JavaScript?

JavaScript is a high-level, interpreted programming language that conforms to the ECMAScript specification. It was initially created to make web pages interactive and is now used for server-side website deployments, non-web browser applications, and much more.

Key Characteristics

  • Client-Side Language: Runs in the browser, enabling dynamic content updates without page reloads
  • Interpreted: Code is executed line by line, without a compilation step
  • Event-Driven: Can respond to user actions like clicks, form submissions, and key presses
  • Object-Oriented: Uses objects to represent and organize data
  • First-Class Functions: Functions can be assigned to variables, passed as arguments, and returned from other functions

Note

JavaScript should not be confused with Java. Although the names are similar, they are very different programming languages with distinct syntax and capabilities.

Adding JavaScript to HTML

There are three main ways to add JavaScript to an HTML document:

1. Inline JavaScript

Add JavaScript directly to HTML elements using the onclick, onload, and other event attributes: 

<button onclick="alert('Hello, World!')">Click Me</button>

2. Internal JavaScript

Add JavaScript within a <script> tag in the HTML document: 

<script>
        function sayHello() {
          alert('Hello, World!');
        }
      </script>
      
      <button onclick="sayHello()">Click Me</button>

3. External JavaScript

Link to an external JavaScript file using the src attribute of the <script> tag: 

<script src="script.js"></script>

Best Practice

It's generally recommended to place JavaScript files at the bottom of the <body> element or use the defer attribute to ensure the HTML content loads before the JavaScript executes: <script src="script.js" defer></script>

Output Methods

JavaScript provides several ways to output data:

1. console.log()

Outputs data to the browser's console, primarily used for debugging: 

console.log('Hello, World!');

2. alert()

Displays data in a dialog box:

alert('Hello, World!');

3. document.write()

Writes directly to the HTML document:

document.write('Hello, World!');

4. innerHTML

Updates the content of an HTML element:

document.getElementById('demo').innerHTML = 'Hello, World!';

Warning

Using document.write() after the HTML document has fully loaded will overwrite all existing content. It should only be used for testing.

JavaScript Fundamentals

Understanding the fundamental concepts of JavaScript is essential for writing effective code. This section covers the core building blocks of the language.

Statements

JavaScript statements are commands that tell the browser what to do. They are composed of values, operators, expressions, keywords, and comments.

Syntax Rules

  • Statements are executed in the order they are written
  • Statements typically end with a semicolon (;)
  • JavaScript is case-sensitive
  • Whitespace is generally ignored
  • Code blocks are enclosed in curly braces {}

Example

// This is a comment
      let x = 5;           // Declare and initialize a variable
      x = x + 10;          // Update variable value
      console.log(x);      // Output to console
      
      if (x > 10) {        // Code block with conditional statement
        console.log("x is greater than 10");
      }

Variables

Variables are containers for storing data values. In JavaScript, there are three ways to declare variables:

var, let, and const

Keyword Scope Reassignment Hoisting Use Case
var Function scope Can be reassigned Hoisted (initialized with undefined) Legacy code (avoid in modern JavaScript)
let Block scope Can be reassigned Hoisted (not initialized) Values that will change
const Block scope Cannot be reassigned Hoisted (not initialized) Values that won't change

Examples

// var (function-scoped)
      var age = 30;
      function printAge() {
        var age = 25;
        console.log(age); // 25
      }
      console.log(age); // 30
      
      // let (block-scoped)
      let count = 10;
      if (true) {
        let count = 5;
        console.log(count); // 5
      }
      console.log(count); // 10
      
      // const (block-scoped, cannot be reassigned)
      const PI = 3.14159;
      // PI = 3.14; // Error: Assignment to constant variable

Modern Best Practice

Use const by default for variables that won't change. Use let for variables that will change. Avoid using var in new code.

Data Types

JavaScript has eight basic data types:

Primitive Data Types

  • String: Text, enclosed in single or double quotes let name = "John";
  • Number: Integers and floating-point numbers let age = 30; let price = 19.99;
  • Boolean: true or false let isActive = true;
  • Undefined: A variable that has been declared but not assigned a value let x;
  • Null: Represents "nothing" or "empty" let empty = null;
  • Symbol: Unique and immutable primitive let id = Symbol("id");
  • BigInt: Integers with arbitrary precision let bigNumber = 1234567890123456789012345678901234567890n;

Non-Primitive (Reference) Data Type

  • Object: Key-value pairs, arrays, functions, dates, etc.

      let numbers = [1, 2, 3];

Functions & Objects

Functions and objects are fundamental building blocks in JavaScript. Functions allow you to create reusable code, while objects let you store collections of data and more complex entities.

Functions

Functions are blocks of code designed to perform a particular task and are executed when called (invoked).

Function Declaration

function greet(name) {
          return "Hello, " + name + "!";
        }
        
        // Calling the function
        let message = greet("John");
        console.log(message); // "Hello, John!"

Function Expression

// Anonymous function assigned to a variable
        const greet = function(name) {
          return "Hello, " + name + "!";
        };
        
        // Calling the function
        console.log(greet("Jane")); // "Hello, Jane!"

Parameters and Arguments

  • Parameters: Variables listed in the function definition
  • Arguments: Values passed to the function when called 
// Function with default parameters (ES6)
        function greet(name = "Guest") {
          return "Hello, " + name + "!";
        }
        
        console.log(greet());        // "Hello, Guest!"
        console.log(greet("Alice")); // "Hello, Alice!"

Return Value

Functions can return a value using the return statement. If no return statement is used, the function returns undefined.

Hoisting

Function declarations are hoisted to the top of their scope, meaning they can be called before they are defined. Function expressions are not hoisted in the same way.

Object Methods

Methods are functions stored as object properties.

Defining Methods

const person = {
          firstName: "John",
          lastName: "Doe",
          // Method
          fullName: function() {
            return this.firstName + " " + this.lastName;
          },
          // Shorthand method (ES6)
          greet() {
            return `Hello, my name is ${this.firstName}`;
          }
        };
        
        console.log(person.fullName()); // "John Doe"
        console.log(person.greet()); // "Hello, my name is John"

The "this" Keyword

The this keyword refers to the object that the method belongs to. In a method, this refers to the owner object.

"this" Context

The value of this depends on how a function is called. In regular functions, it refers to the global object (or undefined in strict mode). In arrow functions, it maintains the this value of the enclosing lexical context.

Object Constructors

Object constructors are functions that create and initialize objects.

Creating an Object Constructor

          this.make = make;
          this.model = model;

Prototypes

All JavaScript objects inherit properties and methods from a prototype. 

// Adding a method to the prototype (more efficient for memory)
        function Car(make, model, year) {
          this.make = make;
          this.model = model;
          this.year = year;
        }
        
        // Add method to prototype (shared across all instances)
        Car.prototype.displayInfo = function() {
          return this.year + " " + this.make + " " + this.model;
        };
        
        const car1 = new Car("Toyota", "Corolla", 2020);
        console.log(car1.displayInfo()); // "2020 Toyota Corolla"

Strings & Arrays

Strings and arrays are fundamental data types in JavaScript. Strings are used to store and manipulate text, while arrays are used to store multiple values in a single variable.

Strings

A string is a sequence of characters enclosed in quotes.

Creating Strings

// Using single quotes
        const singleQuotes = 'Hello, World!';

Array Methods

JavaScript provides many built-in methods for working with arrays.

Adding and Removing Elements

const fruits = ["Apple", "Banana", "Orange"];
            
            // push() - add to the end
            fruits.push("Mango");
            console.log(fruits); // ["Apple", "Banana", "Orange", "Mango"]
            
            // pop() - remove from the end
            const lastFruit = fruits.pop();
            console.log(lastFruit); // "Mango"
            console.log(fruits); // ["Apple", "Banana", "Orange"]
            
            // unshift() - add to the beginning
            fruits.unshift("Strawberry");
            console.log(fruits); // ["Strawberry", "Apple", "Banana", "Orange"]
            
            // shift() - remove from the beginning
            const firstFruit = fruits.shift();
            console.log(firstFruit); // "Strawberry"
            console.log(fruits); // ["Apple", "Banana", "Orange"]

Splicing and Slicing

const fruits = ["Apple", "Banana", "Orange", "Mango", "Kiwi"];
            
            // splice() - add/remove elements from anywhere
            // syntax: array.splice(start, deleteCount, item1, item2, ...)
            
            // Remove elements
            const removed = fruits.splice(2, 2); // Remove 2 elements starting at index 2
            console.log(removed); // ["Orange", "Mango"]
            console.log(fruits); // ["Apple", "Banana", "Kiwi"]
            
            // Add elements
            fruits.splice(1, 0, "Peach", "Lemon"); // Add at index 1, remove 0 elements
            console.log(fruits); // ["Apple", "Peach", "Lemon", "Banana", "Kiwi"]
            
            // Replace elements
            fruits.splice(0, 2, "Pineapple"); // Remove 2 elements starting at index 0, add "Pineapple"
            console.log(fruits); // ["Pineapple", "Lemon", "Banana", "Kiwi"]
            
            // slice() - copy portion of an array (doesn't modify original)
            const citrus = fruits.slice(1, 3); // Elements from index 1 up to (but not including) 3
            console.log(citrus); // ["Lemon", "Banana"]
            console.log(fruits); // Original unchanged: ["Pineapple", "Lemon", "Banana", "Kiwi"]

Other Common Array Methods

// concat() - merge arrays
            const array1 = [1, 2, 3];
            const array2 = [4, 5, 6];
            const merged = array1.concat(array2);
            console.log(merged); // [1, 2, 3, 4, 5, 6]

            const nested = [1, 2, [3, 4, [5, 6]]];
            console.log(nested.flat()); // [1, 2, 3, 4, [5, 6]]
            console.log(nested.flat(2)); // [1, 2, 3, 4, 5, 6]
            
            // toString() - convert array to string
            const numbers = [1, 2, 3, 4];
            console.log(numbers.toString()); // "1,2,3,4"

Array Search

JavaScript provides methods to search for values within arrays.

Finding Elements

const fruits = ["Apple", "Banana", "Orange", "Mango", "Apple"];
            
            // indexOf() - returns the first index of an element
            console.log(fruits.indexOf("Apple")); // 0
            console.log(fruits.indexOf("Apple", 1)); // 4 (start from index 1)
            console.log(fruits.indexOf("Cherry")); // -1 (not found)
            
            // lastIndexOf() - returns the last index of an element
            console.log(fruits.lastIndexOf("Apple")); // 4
            
            // includes() - returns true if array contains the element
            console.log(fruits.includes("Mango")); // true
            console.log(fruits.includes("Grape")); // false

Finding Objects in Arrays

const people = [
              {name: "John", age: 30},
              {name: "Jane", age: 25},
              {name: "Jack", age: 40},
              {name: "Jill", age: 35}
            ];
            
            // find() - returns the first element that satisfies the condition
            const person = people.find(person => person.age > 30);
            console.log(person); // {name: "Jack", age: 40}
            
            // findIndex() - returns the index of the first matching element
            const index = people.findIndex(person => person.name === "Jane");
            console.log(index); // 1
            
            // findLast() and findLastIndex() - search from the end (ES2022)
            const lastPerson = people.findLast(person => person.age > 30);
            console.log(lastPerson); // {name: "Jill", age: 35}
            
            // some() - returns true if at least one element matches
            const hasYoungPerson = people.some(person => person.age < 30);
            console.log(hasYoungPerson); // true
            
            // every() - returns true if all elements match
            const allAdults = people.every(person => person.age >= 18);
            console.log(allAdults); // true

Arrow functions provide a concise syntax for writing function expressions and have some special behavior with this binding.

Basic Syntax

          };

              });

          const person2 = {
            name: "Jane",
            hobbies: ["painting", "swimming", "cooking"],
            showHobbies: function() {
              // 'this' here refers to person2

                // 'this' here still refers to person2
                console.log(`${this.name} enjoys ${hobby}`);
              });
            }
          };

          person1.showHobbies();

          person2.showHobbies();

Arrow Function Limitations

Arrow functions are not suitable for all situations:

  • They can't be used as constructors (with new)
  • They don't have their own arguments object
  • They shouldn't be used for object methods that need to access this from the object
  • They can't be used with yield in generator functions

JavaScript is single-threaded, but it can handle asynchronous operations through callbacks, promises, and async/await. This allows code to execute without blocking while waiting for operations like API calls or timers to complete.

Promises

Promises provide a more elegant solution to handle asynchronous operations and avoid callback hell.

Promise Basics

// Creating a Promise
const myPromise = new Promise((resolve, reject) => {
  // Asynchronous operation
  const success = true;
  
  setTimeout(() => {
    if (success) {
      resolve("Operation completed successfully!");
    } else {
      reject("Operation failed!");
    }
  }, 1000);
});

// Using a Promise
console.log("Promise is pending...");

myPromise
  .then((result) => {
    console.log("Success:", result);
  })
  .catch((error) => {
    console.error("Error:", error);
  })
  .finally(() => {
    console.log("Promise settled (fulfilled or rejected)");
  });

console.log("This code runs before the promise is settled");

Promise States

// Promises have three states:
// 1. Pending: Initial state, neither fulfilled nor rejected
// 2. Fulfilled: Operation completed successfully
// 3. Rejected: Operation failed

// A resolved promise
const resolvedPromise = Promise.resolve("Already resolved");
resolvedPromise.then(value => console.log(value));

// A rejected promise
const rejectedPromise = Promise.reject("Already rejected");
rejectedPromise
  .catch(reason => console.log("Caught:", reason))
  .then(() => console.log("This still runs after catch"));

// A pending promise that never resolves (not very useful)
const pendingPromise = new Promise(() => {
  // Never calls resolve or reject
});

Chaining Promises

// Promise chaining allows sequential async operations
function getUserData(userId) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (userId > 0) {
        resolve({ id: userId, name: "John Doe" });
      } else {
        reject("Invalid user ID");
      }
    }, 1000);
  });
}

function getUserPosts(user) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve([
        { id: 1, userId: user.id, title: "Post 1" },
        { id: 2, userId: user.id, title: "Post 2" }
      ]);
    }, 1000);
  });
}

function getPostComments(post) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve([
        { id: 1, postId: post.id, text: "Great post!" },
        { id: 2, postId: post.id, text: "I enjoyed reading this" }
      ]);
    }, 1000);
  });
}

// Promise.all - Wait for all promises to resolve
const promise1 = Promise.resolve("One");
const promise2 = new Promise((resolve) => setTimeout(() => resolve("Two"), 1000));

  })

  });

Promise.race([
  new Promise(resolve => setTimeout(() => resolve("Fast"), 500)),
  new Promise(resolve => setTimeout(() => resolve("Slow"), 1000))
])
  .then(value => console.log("Promise.race winner:", value))
  .catch(error => console.error("Race error:", error));

// Promise.allSettled - Wait for all promises to settle (ES2020)
Promise.allSettled([
  Promise.resolve("Success"),
  Promise.reject("Failure"),
  new Promise(resolve => setTimeout(() => resolve("Delayed"), 1000))
])
  .then(results => {
    console.log("Promise.allSettled results:");
    results.forEach((result, index) => {
      console.log(`Promise ${index + 1}:`, result);
    });
  });

// Promise.any - Resolves as soon as one promise fulfills (ES2021)
Promise.any([
  new Promise((resolve, reject) => setTimeout(() => reject("Rejected 1"), 1000)),
  new Promise((resolve) => setTimeout(() => resolve("Resolved 1"), 2000)),
  new Promise((resolve) => setTimeout(() => resolve("Resolved 2"), 1500))
])
  .then(value => console.log("Promise.any result:", value))
  .catch(error => console.error("All promises were rejected:", error));

Async/Await

Async/await is syntactic sugar built on top of promises, making asynchronous code look and behave more like synchronous code.

Basic Syntax

// async functions always return a promise
async function greet() {
  return "Hello, world!";
}

// This is equivalent to:
function greetWithPromise() {
  return Promise.resolve("Hello, world!");
}

// Using an async function
greet().then(message => console.log(message));

// await keyword can only be used inside async functions
async function fetchData() {
  console.log("Fetching data...");
  
  // await pauses execution until the promise settles
  const data = await new Promise(resolve => {
    setTimeout(() => resolve("Data received"), 1000);
  });
  
  console.log(data);
  return "Operation complete";
}

fetchData().then(result => console.log(result));

Error Handling

// Using try/catch with async/await
async function fetchWithErrorHandling() {
  try {
    console.log("Starting operation...");
    
    const result = await new Promise((resolve, reject) => {
      const success = false; // Change to true to see success path
      
      setTimeout(() => {
        if (success) {
          resolve("Operation successful");
        } else {
          reject(new Error("Something went wrong"));
        }
      }, 1000);
    });
    
    console.log("Result:", result);
    return result;
  } catch (error) {
    console.error("Caught an error:", error.message);
    return "Recovered from error";
  } finally {
    console.log("Operation complete (success or failure)");
  }
}

function calculateTotalPrice(items, taxRate, discountCode) {
  // Base price before tax and discounts
  const subtotal = items.reduce((sum, item) => sum + item.price, 0);
  console.log("Subtotal:", subtotal);
  
  // Apply discount if valid
  let discountAmount = 0;
  if (discountCode === "SAVE20") {
    discountAmount = subtotal * 0.2;
  }
  console.log("Discount amount:", discountAmount);
  
  // Calculate tax on the discounted amount
  const taxableAmount = subtotal - discountAmount;
  const taxAmount = taxableAmount * taxRate;
  console.log("Tax amount:", taxAmount);
  
  // Final total

}
