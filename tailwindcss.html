Introduction to Tailwind CSS

Tailwind CSS is a utility-first CSS framework that allows you to build custom designs without leaving your HTML. Unlike other CSS frameworks that provide pre-designed components, Tailwind gives you low-level utility classes that let you build completely custom designs.

What is Tailwind CSS?

Tailwind CSS is a highly customizable, low-level CSS framework that provides you with all the building blocks you need to build bespoke designs without any opinionated styles you have to fight to override.

Key Characteristics

  • Utility-First: Composed of small, single-purpose utility classes that can be composed to build any design
  • Responsive: Built-in responsive modifiers make it easy to build responsive interfaces
  • Component-Friendly: Utilities can be extracted into reusable components with tools like @apply
  • Customizable: Tailwind is designed to be customized to your project's needs
  • Production-Ready: Includes tools to optimize CSS for production by removing unused styles

How Tailwind Works

Unlike traditional CSS frameworks like Bootstrap or Foundation, Tailwind doesn't provide pre-built components. Instead, it provides utility classes that you can combine to create your own designs directly in your HTML.

Note

Tailwind CSS is highly optimized for production through its built-in PurgeCSS integration, which removes unused CSS, resulting in very small file sizes.

Benefits of Tailwind CSS

Tailwind CSS offers numerous advantages over traditional CSS frameworks and methodologies:

Advantages of Tailwind CSS

  • Faster Development: Build custom interfaces without writing custom CSS
  • Consistency: Predefined spacing, colors, and other design tokens ensure consistency
  • Responsive by Default: Built-in responsive variants make building for different screen sizes easy
  • Maintainability: Changes can be made directly in the HTML without hunting through stylesheets
  • Lower CSS File Size: PurgeCSS integration removes unused styles for smaller production builds
  • Reduced Decision Fatigue: Well-thought-out defaults for colors, spacing, etc.
  • No Naming Problems: Avoid coming up with class names for every element

Comparison with Traditional CSS Workflows

Traditional CSS Tailwind CSS
Write custom CSS for each component Use utility classes directly in HTML
Create and maintain class naming system Use pre-defined utility class names
CSS file size grows as project grows CSS file size can remain small with proper purging
Context switching between HTML and CSS files Stay in HTML for both structure and styling
Need to create media queries for responsive design Use built-in responsive variants (e.g., md:flex)

Tailwind vs Other CSS Frameworks

Understanding how Tailwind CSS compares to other popular CSS frameworks helps in choosing the right tool for your project.

Tailwind CSS vs Bootstrap

Feature Tailwind CSS Bootstrap
Approach Utility-first Component-based
Customization Highly customizable Less customizable without overrides
Design Opinion No predefined look Strong visual identity
Learning Curve Steeper initially, simpler later Easier to start, harder to customize
File Size Potentially smaller with purging Larger, especially with jQuery

When to Choose Tailwind CSS

Tailwind CSS is particularly well-suited for:

  • Custom designs that don't follow a specific design system
  • Projects where you want maximum design flexibility
  • Teams that prefer working primarily in HTML/JSX
  • Projects where front-end performance is critical
  • Applications that need to be highly responsive across devices

Important Consideration

Tailwind's utility-first approach can lead to verbose HTML if not managed properly. For complex components, consider extracting repeated utility patterns into component classes using @apply or a component-based JavaScript framework.

Installation & Setup

Getting started with Tailwind CSS involves installing the framework, configuring it to your project's needs, and setting up the build process. This section covers different approaches to integrating Tailwind into your projects.

Install Tailwind CSS

There are several ways to install Tailwind CSS, depending on your project setup and requirements.

Using npm (Recommended)

The most common way to install Tailwind is through npm:

# Install Tailwind CSS, PostCSS and autoprefixer
npm install -D tailwindcss postcss autoprefixer

# Generate the configuration files
npx tailwindcss init -p

This creates two configuration files:

  • tailwind.config.js - Tailwind CSS configuration
  • postcss.config.js - PostCSS configuration

Framework-specific Guides

Tailwind offers official guides for installing with popular frameworks:

  • React/Next.js: Additional setup for React applications
  • Vue/Nuxt: Integration with Vue.js ecosystem
  • Angular: Setup for Angular projects
  • Laravel: Built-in support through Laravel Mix

Using Play CDN (for prototyping only)

For quick prototyping, you can use the Play CDN:

<!-- Add to your HTML head -->
<script src="https://cdn.tailwindcss.com"></script>

Important

The Play CDN should only be used for development and demos, not for production. It doesn't include important optimizations like minimizing file size.

Configuration File

The tailwind.config.js file allows you to customize virtually every aspect of Tailwind CSS.

Basic Configuration Structure

// tailwind.config.js
/** @type {import('tailwindcss').Config} */
module.exports = {
  content: [
    "./src/**/*.{html,js,jsx,ts,tsx}",
    "./public/index.html",
  ],
  theme: {
    extend: {},
  },
  plugins: [],
}

Key Configuration Options

Option Description
content Files to scan for class names to include in your CSS
theme Customizations for colors, spacing, typography, etc.
plugins Additional Tailwind plugins to use
prefix Optional prefix for all utilities (e.g., 'tw-')
important Control the CSS importance level
darkMode Configure dark mode strategy ('media' or 'class')

Customizing Theme

The theme section is where you customize Tailwind's design tokens: 

module.exports = {
  // ...
  theme: {
    // Override defaults
    colors: {
      blue: {
        light: '#85d7ff',
        DEFAULT: '#1fb6ff',
        dark: '#009eeb',
      },
      // ...
    },
    // Extend defaults
    extend: {
      spacing: {
        '72': '18rem',
        '84': '21rem',
        '96': '24rem',
      },
      borderRadius: {
        '4xl': '2rem',
      }
    }
  }
}

Tip

Use theme.extend to add to Tailwind's defaults instead of theme which overrides them entirely.

Build Process

Tailwind CSS requires a build step to generate your CSS file, especially for production environments.

Setting Up Source CSS File

Create a CSS file (e.g., input.css) with the Tailwind directives: 

/* input.css */
@tailwind base;
@tailwind components;
@tailwind utilities;

Processing CSS with PostCSS

Configure PostCSS to process your CSS:

// postcss.config.js
module.exports = {
  plugins: {
    tailwindcss: {},
    autoprefixer: {},
  }
}

Build Scripts

Add build scripts to your package.json:

{
  "scripts": {
    "build:css": "tailwindcss -i ./src/input.css -o ./dist/output.css",
    "watch:css": "tailwindcss -i ./src/input.css -o ./dist/output.css --watch"
  }
}

Optimizing for Production

For production, you'll want to minimize your CSS file:

{
  "scripts": {
    "build:css": "tailwindcss -i ./src/input.css -o ./dist/output.css --minify"
  }
}

Tailwind automatically purges unused styles when building for production based on your content configuration.

Integration with Framework Build Tools

Most modern JavaScript frameworks have build tools that can integrate with Tailwind:

  • Vite: Configure PostCSS in vite.config.js
  • Webpack: Use with postcss-loader
  • Next.js: Built-in support with CSS modules

CDN (Quick Start)

For quick prototyping or small projects, you can use Tailwind CSS directly from a CDN without any build step.

Using the Play CDN

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Tailwind CSS Demo</title>
  <script src="https://cdn.tailwindcss.com"></script>
  <!-- Optional: Configure Tailwind directly in HTML -->
  <script>
    tailwind.config = {
      theme: {
        extend: {
          colors: {
            clifford: '#da373d',
          }
        }
      }
    }
  </script>
  <!-- Optional: Add custom styles -->
  <style type="text/tailwindcss">
    @layer utilities {
      .content-auto {
        content-visibility: auto;
      }
    }
  </style>
</head>
<body>
  <div class="container mx-auto px-4 py-8">
    <h1 class="text-3xl font-bold text-clifford">Hello World</h1>
  </div>
</body>
</html>

Limitations of CDN Approach

  • No file size optimization through PurgeCSS
  • Limited customization options
  • No tree-shaking of unused styles
  • Not suitable for production environments
  • Limited browser caching benefits

Best Practice

Use the CDN approach for small demos, prototypes, or learning Tailwind CSS. For production projects, always use the proper installation with npm and build process to benefit from optimizations.

Responsive Design

Tailwind CSS makes responsive design straightforward with its mobile-first approach and intuitive breakpoint system. This section explores how to create responsive layouts using Tailwind's responsive utilities.

Breakpoints

Tailwind uses a mobile-first approach with responsive breakpoints that allow you to build designs that adapt to different screen sizes.

Default Breakpoints

Breakpoint Prefix Minimum Width CSS
sm 640px @media (min-width: 640px) { ... }
md 768px @media (min-width: 768px) { ... }
lg 1024px @media (min-width: 1024px) { ... }
xl 1280px @media (min-width: 1280px) { ... }
2xl 1536px @media (min-width: 1536px) { ... }

Customizing Breakpoints

You can customize the default breakpoints in your tailwind.config.js file: 

// tailwind.config.js
        module.exports = {
          theme: {
            screens: {
              'sm': '640px',
              'md': '768px',
              'lg': '1024px',
              'xl': '1280px',
              '2xl': '1536px',
              // Custom breakpoints
              'tablet': '640px',
              'laptop': '1024px',
              'desktop': '1280px',
            }
          }
        }

Note

By defining custom breakpoints, you'll replace the default ones unless you use theme.extend.screens instead.

Responsive Utilities

Tailwind allows you to apply different styles at different breakpoints using the responsive prefix syntax.

Basic Responsive Syntax

<!-- Width responsive to screen size -->
        <div class="w-full md:w-1/2 lg:w-1/3">
          <!-- Full width on mobile, half width on medium screens, one-third on large screens -->
        </div>
        
        <!-- Responsive text alignment -->
        <p class="text-center md:text-left lg:text-right">
          <!-- Centered on mobile, left-aligned on medium screens, right-aligned on large screens -->
        </p>
        
        <!-- Responsive display -->
        <div class="hidden md:block">Only visible on md screens and up</div>
        <div class="block md:hidden">Only visible on screens smaller than md</div>

Responsive Flexbox/Grid Layout

<!-- Responsive direction -->
        <div class="flex flex-col md:flex-row">
          <!-- Stacked on mobile, side-by-side on medium screens and up -->
          <div>Item 1</div>
          <div>Item 2</div>
        </div>
        
        <!-- Responsive grid columns -->
        <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3">
          <!-- 1 column on mobile, 2 columns on medium, 3 columns on large -->
          <div>Item 1</div>
          <div>Item 2</div>
          <div>Item 3</div>
        </div>

Responsive Spacing

<!-- Responsive padding -->
        <div class="p-4 md:p-8 lg:p-12">
          <!-- Different padding at different screen sizes -->
        </div>
        
        <!-- Responsive margin -->
        <div class="mt-4 md:mt-8 lg:mt-12">
          <!-- Different top margin at different screen sizes -->
        </div>
        
        <!-- Responsive gap -->
        <div class="grid grid-cols-2 gap-2 md:gap-4 lg:gap-6">
          <!-- Different gap at different screen sizes -->
        </div>

Responsive Typography

<!-- Responsive font size -->
        <h1 class="text-2xl md:text-3xl lg:text-4xl">
          Responsive Heading
        </h1>
        
        <!-- Responsive text styling -->
        <p class="font-normal md:font-medium text-base md:text-lg">
          Text that changes size and weight
        </p>

Mobile-First Approach

Tailwind implements a mobile-first breakpoint system, which means that unprefixed utilities apply to all screen sizes, while prefixed utilities apply only at the specified breakpoint and above.

Understanding Mobile-First

With a mobile-first approach, you should:

  1. Design for mobile screens first
  2. Then add responsive prefixes to adapt the design for larger screens 
<!-- Example of mobile-first thinking -->
        <div class="
          w-full          /* Mobile: full width */
          md:w-1/2        /* Tablet: half width */
          lg:w-1/3        /* Desktop: one-third width */
          p-4             /* All sizes: padding of 4 (1rem) */
          md:p-6          /* Tablet & up: padding of 6 (1.5rem) */
          text-sm         /* All sizes: small text */
          md:text-base    /* Tablet & up: base text size */
          lg:text-lg      /* Desktop & up: large text */
        ">
          This element adapts to different screen sizes
        </div>

Building Complex Responsive Layouts

<!-- Complex responsive layout example -->
        <div class="container mx-auto px-4 py-8">
          <!-- Header section -->
          <header class
          =
    "flex flex-col md:flex-row items-center justify-between mb-8 gap-4"
  >
    <div class="text-center md:text-left">
      <h1 class="text-3xl font-bold md:text-4xl lg:text-5xl">Site Title</h1>
      <p class="text-gray-600 mt-2">Site description goes here</p>
    </div>
    <nav>
      <ul class="flex gap-4 flex-wrap justify-center md:justify-end">
        <li><a href="#" class="text-blue-600 hover:text-blue-800">Home</a></li>
        <li><a href="#" class="text-blue-600 hover:text-blue-800">About</a></li>
        <li><a href="#" class="text-blue-600 hover:text-blue-800">Services</a></li>
        <li><a href="#" class="text-blue-600 hover:text-blue-800">Contact</a></li>
      </ul>
    </nav>
  </header>
  
  <!-- Main content section -->
  <main class="mt-8">
    <div class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-6">
      <!-- Repeated cards -->
      <div class="bg-white rounded-lg shadow-md p-6">
        <h2 class="text-xl font-semibold mb-4">Card Title</h2>
        <p class="text-gray-600">Card content goes here.</p>
      </div>
      <!-- More cards... -->
    </div>
  </main>
</div>

Best Practice

When building responsive layouts, start by designing for mobile devices first, then progressively enhance for larger screens. This approach typically results in cleaner and more maintainable code.

Extracting Button Styles

To avoid repeating button styles, you can create custom button components or use the @apply directive in a CSS file: 

/* In your CSS file */
.btn {
  @apply font-bold py-2 px-4 rounded;
}
.btn-blue {
  @apply bg-blue-500 text-white;
}
.btn-blue:hover {
  @apply bg-blue-700;
}
.btn-outline {
  @apply bg-transparent text-blue-700 border border-blue-500;
}
.btn-outline:hover {
  @apply bg-blue-500 text-white border-transparent;
}

Then use these classes in your HTML:

<button class="btn btn-blue">Primary Button</button>
<button class="btn btn-outline">Secondary Button</button>

Advanced Styling

Tailwind CSS provides several advanced features that allow you to extend its functionality, create custom utilities, and implement complex styling requirements. This section explores these advanced capabilities.

Custom Utilities

While Tailwind provides a comprehensive set of built-in utilities, you can create your own custom utilities to match your project's specific needs.

Using @layer to Add Custom Utilities

The @layer directive allows you to add your own custom styles to Tailwind's utility layers: 

/* In your CSS file */
        @tailwind base;
        @tailwind components;
        @tailwind utilities;
        
        @layer utilities {
          .text-shadow {
            text-shadow: 2px 2px 4px rgba(0, 0, 0, 0.5);
          }
          
          .text-shadow-sm {
            text-shadow: 1px 1px 2px rgba(0, 0, 0, 0.5);
          }
          
          .text-shadow-lg {
            text-shadow: 4px 4px 8px rgba(0, 0, 0, 0.5);
          }
          
          .text-shadow-none {
            text-shadow: none;
          }
        }

Using the Plugin API

For more complex utilities, you can use Tailwind's plugin API: 

// tailwind.config.js
        const plugin = require('tailwindcss/plugin')
        
        module.exports = {
          // ...
          plugins: [
            plugin(function({ addUtilities, theme }) {
              const newUtilities = {
                '.text-shadow-sm': {
                  textShadow: '1px 1px 2px rgba(0, 0, 0, 0.5)',
                },
                '.text-shadow': {
                  textShadow: '2px 2px 4px rgba(0, 0, 0, 0.5)',
                },
                '.text-shadow-lg': {
                  textShadow: '4px 4px 8px rgba(0, 0, 0, 0.5)',
                },
                '.text-shadow-none': {
                  textShadow: 'none',
                },
              }
        
              addUtilities(newUtilities, ['responsive', 'hover'])
            }),
          ],
        }

This approach makes the utilities responsive and adds hover variants.

Creating Responsive Variants

Custom utilities can include responsive variants:

<div class="text-shadow md:text-shadow-lg lg:text-shadow-none">
          Text with responsive shadow styles
        </div>

Note

When adding custom utilities, ensure they follow Tailwind's naming conventions for consistency. Also, only add utilities that are used frequently enough to justify their inclusion.

// tailwind.config.js

Creating Custom Variants

You can create custom variants for specific use cases:

// tailwind.config.js
        const plugin = require('tailwindcss/plugin')
        
        module.exports = {
          // ...
          plugins: [
            plugin(function({ addVariant }) {
              // Add a `parent-hover` variant

            })

        }

/* In your CSS file */

            @apply py-2 px-4 rounded focus:outline-none focus:shadow-outline font-medium;

Dark Mode

Tailwind CSS provides built-in support for dark mode, allowing you to create designs that adapt to user preferences.

Configuring Dark Mode

First, you need to configure the dark mode strategy in your tailwind.config.js file: 

// tailwind.config.js
        module.exports = {
          darkMode: 'media', // or 'class'
          // ...
        }
  • 'media': Responds to the system preference (prefers-color-scheme)
  • 'class': Manually controlled with the 'dark' class on an ancestor element

Using Dark Mode Variants

Use the dark: prefix to apply styles only in dark mode: 

<div class="bg-white dark:bg-gray-800 text-gray-900 dark:text-white">
          <h1 class="text-gray-900 dark:text-white">This heading adapts to dark mode</h1>
          <p class="text-gray-600 dark:text-gray-300">This text is gray in light mode and light gray in dark mode.</p>
          <button class="bg-blue-500 hover:bg-blue-600 dark:bg-blue-600 dark:hover:bg-blue-700 text-white">
            Button
          </button>
        </div>

Manual Dark Mode Toggle

If using 'class' strategy, you'll need JavaScript to toggle the dark mode: 

<!-- Toggle Button -->
        <button id="dark-mode-toggle" class="p-2 rounded-md hover:bg-gray-200 dark:hover:bg-gray-700">
          <svg class="w-6 h-6 hidden dark:block" fill="none" stroke="currentColor" viewBox="0 0 24 24" xmlns="http://www.w3.org/2000/svg">
            <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M12 3v1m0 16v1m9-9h-1M4 12H3m15.364 6.364l-.707-.707M6.343 6.343l-.707-.707m12.728 0l-.707.707M6.343 17.657l-.707.707M16 12a4 4 0 11-8 0 4 4 0 018 0z"></path>
          </svg>
          <svg class="w-6 h-6 block dark:hidden" fill="none" stroke="currentColor" viewBox="0 0 24 24" xmlns="http://www.w3.org/2000/svg">
            <path stroke-linecap="round" stroke-linejoin="round" stroke-width="2" d="M20.354 15.354A9 9 0 018.646 3.646 9.003 9.003 0 0012 21a9.003 9.003 0 008.354-5.646z"></path>
          </svg>
        </button>
        
        <script>
          // Check for saved theme preference or use system preference
          if (localStorage.theme === 'dark' || (!('theme' in localStorage) && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
            document.documentElement.classList.add('dark')
          } else {
            document.documentElement.classList.remove('dark')
          }
        
          // Toggle dark mode on button click
          document.getElementById('dark-mode-toggle').addEventListener('click', function() {
            if (document.documentElement.classList.contains('dark')) {
              document.documentElement.classList.remove('dark')
              localStorage.theme = 'light'
            } else {
              document.documentElement.classList.add('dark')
              localStorage.theme = 'dark'
            }
          })
        </script>

Dark Mode Best Practices

  • Don't just invert colors; consider contrast and readability
  • Reduce brightness and saturation of images and illustrations
  • Soften whites to reduce eye strain (use gray-50/gray-100 instead of pure white)
  • Use semantic color variables instead of literal colors when possible

Creating Custom Plugins

Create your own plugins for project-specific functionality:

// tailwind.config.js
        const plugin = require('tailwindcss/plugin')
        
        module.exports = {
          // ...
          plugins: [
            plugin(function({ addComponents, theme }) {
              const buttons = {
                '.btn': {
                  padding: `${theme('spacing.2')} ${theme('spacing.4')}`,
                  borderRadius: theme('borderRadius.md'),
                  fontWeight: theme('fontWeight.medium'),
                  '&:focus': {
                    outline: 'none',
                    boxShadow: `0 0 0 3px ${theme('colors.blue.300')}`,
                  },
                },
                '.btn-primary': {
                  backgroundColor: theme('colors.blue.500'),
                  color: theme('colors.white'),
                  '&:hover': {
                    backgroundColor: theme('colors.blue.600'),
                  },
                },
                '.btn-secondary': {
                  backgroundColor: theme('colors.gray.200'),
                  color: theme('colors.gray.800'),
                  '&:hover': {
                    backgroundColor: theme('colors.gray.300'),
                  },
                },
              }
        
              addComponents(buttons)
            }),
          ],
        }

Available Plugin Hooks

Hook Description
addBase Add base styles (global resets, etc.)
addComponents Add component classes
addUtilities Add utility classes
addVariant Add custom variants
matchUtilities Add utilities that map to values in the theme
matchComponents Add components that map to values in the theme

Best Practice

When creating custom plugins, consider publishing them as separate npm packages if they might be useful to others. This promotes code reuse and community contribution.

Best Practices

As with any tool, there are effective ways to use Tailwind CSS. This section covers best practices for organizing your code, optimizing performance, and implementing common UI patterns.

Organization

Keeping your Tailwind projects organized is essential for maintainability, especially as projects grow in size.

Managing Class Names

Long lists of utility classes can become difficult to read. Here are some strategies to manage them: 

<!-- Bad: Hard to read, edit, and maintain -->
        <button class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded focus:outline-none focus:shadow-outline transform hover:scale-105 transition-transform duration-200 ease-in-out shadow-lg">
          Click Me
        </button>
        
        <!-- Better: Classes grouped by purpose with line breaks -->
        <button class="
          bg-blue-500 hover:bg-blue-700 
          text-white font-bold 
          py-2 px-4 
          rounded 
          focus:outline-none focus:shadow-outline 
          transform hover:scale-105 transition-transform duration-200 ease-in-out 
          shadow-lg
        ">
          Click Me
        </button>

Extracting Components

For repeated patterns, extract them into components using one of these methods:

1. Using @apply in CSS:

/* In your CSS file */
        @layer components {
          .btn-primary {
            @apply bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded focus:outline-none focus:shadow-outline transform hover:scale-105 transition-transform duration-200 ease-in-out shadow-lg;
          }
        }

2. Using template partials or components in your framework:

// React Component Example
        function Button({ children, ...props }) {
          return (
            <button 
              className="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded focus:outline-none focus:shadow-outline transform hover:scale-105 transition-transform duration-200 ease-in-out shadow-lg"
              {...props}
            >
              {children}
            </button>
          );
        }

Project Structure

Organize your Tailwind CSS project with a clear structure:

src/
          ├── styles/
          │   ├── main.css           # Main CSS file with Tailwind imports
          │   ├── base/              # Base styles and overrides
          │   ├── components/        # Component styles using @apply
          │   └── utilities/         # Custom utilities
          ├── components/            # React/Vue/template components
          ├── layouts/               # Layout templates
          ├── pages/                 # Page components/templates
          └── tailwind.config.js     # Tailwind configuration

Tip

For larger projects, consider using a component-based approach with a design system that documents your common components and their usage.

Performance

Tailwind CSS is designed with performance in mind, but there are additional steps you can take to optimize your project.

Purging Unused Styles

Tailwind can generate thousands of utility classes, but most projects only use a small subset. Configure content purging to remove unused styles: 

// tailwind.config.js
        module.exports = {
          // Content paths to scan for class names
          content: [
            './pages/**/*.{js,jsx,ts,tsx}',
            './components/**/*.{js,jsx,ts,tsx}',
            './layouts/**/*.{js,jsx,ts,tsx}',
            './public/**/*.html',
          ],
          // ...
        }

Just-in-Time Mode

Tailwind's JIT mode generates styles on-demand, which leads to faster build times and smaller CSS files: 

// tailwind.config.js
        module.exports = {
          // Just-in-Time mode is now enabled by default in Tailwind CSS v3.0+
          // No additional configuration needed
        }

Minification

Ensure your production CSS is minified:

# Using PostCSS CLI
        postcss styles/main.css -o dist/styles.css --env production
        
        # Or with package.json script
        "scripts": {
          "build:css": "NODE_ENV=production tailwindcss -i ./src/styles/main.css -o ./dist/styles.css --minify"
        }

Common Performance Pitfalls

  • Not configuring the content array properly: Missing files means needed classes might be purged
  • Dynamic class names: Class names constructed using string concatenation may be missed by the purge
  • Using arbitrary values excessively: Each arbitrary value creates a new CSS rule

Handling Dynamic Classes

When using dynamic class names, safelist the classes that might be missed by the purge: 

// tailwind.config.js
        module.exports = {
          content: [
            './src/**/*.{html,js,jsx,ts,tsx}',
          ],
          safelist: [
            'bg-red-500',
            'bg-green-500',
            'bg-blue-500',
            {
              pattern: /bg-(red|green|blue)-(100|200|300|400|500|600|700|800|900)/,
            },
          ],
          // ...
        }
